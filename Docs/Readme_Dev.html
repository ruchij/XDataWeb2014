The developer mode allows for setting up the tool in an IDE such as Eclipse.

  1. Download XData.tar.gz and extract it using the following command
    	tar zxfv XData.tar.gz
  2. Download json-simple.jar and place it in the WebContent/WEB-INF/lib folder.
  3. Download and install Postgresql. Setup the database.
  4. Install Eclipse EE Edition (Juno or better) and JDK 1.6/1.7. Set up the appropriate environment variables and try running SimpleTest.java 
    public class SimpleTest {
    public static void main(String[] args) {
	System.out.println("This works!");
    }
}
  5. Open Eclipse and click on Window->Preferences->Java->Installed JREs. Make sure JDK 1.6 or more is there. If it not there, click on Add to add it and then mark it as the default JRE.
  6. Click on File->Import and under Java->Existing Projects into workspace. Browse to the root of XData directory and click Add.
  7. Go to Window->Show View->Others and select Servers. Click on add a new server. Select Tomcat 7.0 and set it up. In the end, add XData to the Tomcat Server projects and close the window.
  8. In tool.jsp set the debug flag to 1, i.e. debug = 1, to run the project without the need of integrating with Moodle.
  9. Build your project, and open the following URL in the browser 
    localhost:8080/XDataWeb2013/Empty.html
